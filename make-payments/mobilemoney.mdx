With the Xente API, you can send mobile money in three easy steps.
## Step 1 - Verify mobile money number

To ensure your sending money to the correct mobile money number, please first verify the person your sending funds to

<CodeGroup>

```bash Java

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.net.HttpURLConnection;
import java.net.URL;

public class Main {
    public static void main(String[] args) throws IOException {
        // Set the URL and request method
        String url = "https://api.xente.co/people/";
        URL obj = new URL(url);
        HttpURLConnection con = (HttpURLConnection) obj.openConnection();
        con.setRequestMethod("GET");

        // Set the request headers
        con.setRequestProperty("Content-Type", "application/json");
        con.setRequestProperty("Authorization", "Token <token>");

        // Set the request body (parameters)
        String requestBody = "{\"id\": \"256770123456\", \"name\": \"Halle Berry\", \"source\": \"MTNMOMOUG\"}";

        // Send the request
        con.setDoOutput(true);
        con.getOutputStream().write(requestBody.getBytes());

        // Get the response
        int responseCode = con.getResponseCode();
        BufferedReader in = new BufferedReader(new InputStreamReader(con.getInputStream()));
        String inputLine;
        StringBuilder response = new StringBuilder();
        while ((inputLine = in.readLine()) != null) {
            response.append(inputLine);
        }
        in.close();

        // Print the response
        System.out.println("Response Code: " + responseCode);
        System.out.println("Response Body: " + response.toString());
    }
}
```

```bash PHP
        'Content-Type: application/json',

    );

        "id" => "256770123456",
        "name" => "Halle Berry",
        "source" => "MTNMOMOUG"
    );
    
    $ch = curl_init($url);
    curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
    curl_setopt($ch, CURLOPT_CUSTOMREQUEST, 'GET');
    curl_setopt($ch, CURLOPT_HTTPHEADER, $headers);
    curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($data));
    
    $response = curl_exec($ch);
    
    if ($response === false) {
        $error = curl_error($ch);
        curl_close($ch);
        return "Error: " . $error;
    }
    
    curl_close($ch);
    return $response;
}
```

## Step 2 - POST mobile money transaction

You POST a mobile money transaction as below  

<CodeGroup>

```bash cURL

curl --location --request POST 'https://api.xente.co/transactions' \
--header 'Content-Type: application/json' \
--header 'Authorization: Token <token>' \
--data-raw '{
    "clientreference_id": 67890,
    "batch_id": 54321,
    "account_number": "9876543210",
    "name": "John Doe",
    "currency": "UGX",
    "amount": 1000,
    "transaction_type": "debit",
    "product": {
        "id": "MTNMOMOUG",
        "phone_number": "25677123456",
        "amount": "1000",
        "message": "message to be received with funds"
    },
    "beneficiary": {
        "id": "123123",
        "names": "John Rambo",
        "address": "Some address",
        "other_info": "Other info on beneficiary e.g. team, department"
    }
}'



```

```bash Java
public class Main {
    public static void main(String[] args) throws Exception {
        String url = "https://api.xente.co/transactions";
        String token = "<token>";

        String requestBody = "{\n" +
                "    \"clientreference_id\": 67890,\n" +
                "    \"batch_id\": 54321,\n" +
                "    \"account_number\": \"9876543210\",\n" +

                "    \"currency\": \"UGX\",\n" +
                "    \"amount\": 1000,\n" +
                "    \"transaction_type\": \"debit\",\n" +
                "    \"product\": {\n" +
                "        \"id\": \"MTNMOMOUG\",\n" +

                "        \"amount\": \"1000\",\n" +
                "        \"message\": \"message to be received with funds\"\n" +
                "    },\n" +
                "    \"beneficiary\": {\n" +
                "        \"id\": \"123123\",\n" +
                "        \"names\": \"John Rambo\",\n" +
                "        \"address\": \"Some address\",\n" +
                "        \"other_info\": \"Other info on beneficiary e.g. team, department\"\n" +
                "    }\n" +
                "}";

        URL apiUrl = new URL(url);
        HttpURLConnection connection = (HttpURLConnection) apiUrl.openConnection();
        connection.setRequestMethod("POST");
        connection.setRequestProperty("Content-Type", "application/json");
        connection.setRequestProperty("Authorization", "Token " + token);
        connection.setDoOutput(true);

        try (OutputStream outputStream = connection.getOutputStream()) {
            byte[] input = requestBody.getBytes(StandardCharsets.UTF_8);
            outputStream.write(input, 0, input.length);
        }

        int responseCode = connection.getResponseCode();
        System.out.println("Response Code: " + responseCode);

        connection.disconnect();
    }
}
```

</CodeGroup>


| Key                         | Type                   | Description
| --------------------------- | ---------------------- | -------------
| client_reference_id         | string                 | A unique ID from your system to help you identify the transaction
| batch_id                    | string                 | A unique ID to help you identify the batch. This is useful if the transaction is part of a bulk payment
| account_number              | string                 | Your business account number from which funds will be debited
| currency                    | string                 | The currency of your account
| amount                      | number                 | The amount to be debited from your account
| transaction_type               | string                 | Put value "debit" to send mobile money as this debits funds from your account
| product.id                  | string                 | Put the value for Mobile Money Product IDs. You can find IDs under products in the portal
| product.amount              | number                 | The amount of mobile money to be sent
| product.message             | string                 | The message that the beneficiary will receive
| beneficiary.id              | string                 | The unique ID of the beneficiary in your system
| beneficiary.names           | string                 | The names of the beneficiary
| beneficiary.other_information | string                | Other information about the beneficiary





## Step 3 - Receive Instant Payment Notification (IPN)

All transactions on Xente are asynchronous. Therefore, after you POST a mobile money transaction, you need to wait for a success (or failure) response using webhooks called Instant Payment Notifications (IPN).

Learn more about **[Instant Payment Notifications](/start/ipn)**.
